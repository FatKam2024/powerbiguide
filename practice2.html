Practice Questions: Set 2 (Advanced)

Advanced Questions

Test your understanding of advanced Power BI topics with these questions:

  1. How do you create a relationship between two tables in Power BI?
  2. Explain the difference between a calculated column and a measure in DAX.
  3. What is the purpose of the CALCULATE function in DAX?
  4. How do you perform a year-to-date calculation using DAX?
  5. What are the steps to publish a report to Power BI Service?
  6. How do you implement Row-Level Security (RLS) in Power BI?
  7. What is the purpose of the RELATED function in DAX?
  8. How do you create a custom calendar table in Power BI?
  9. Explain the concept of filter context in DAX.
  10. How do you use parameters in Power BI reports?

Answers

  1. Go to the Model view, drag fields between tables to create a relationship, or use the Manage Relationships dialog.
  2. A calculated column is computed row by row and stored in the table, while a measure is computed on demand and used in visualizations.
  3. The CALCULATE function modifies the filter context of a calculation, allowing for complex aggregations and filtering.
  4. Use the TOTALYTD function in DAX, e.g., TOTALYTD(SUM(Sales[Amount]), 'Date'[Date]).
  5. Open the report in Power BI Desktop, click "Publish," select your workspace, and click "Select."
  6. Create roles in Power BI Desktop, define DAX filters for each role, then manage security in Power BI Service.
  7. RELATED is used to retrieve a value from a related table in a many-to-one relationship.
  8. Create a new table using DAX, typically with the CALENDAR or CALENDARAUTO function, then add calculated columns for date attributes.
  9. Filter context refers to the set of filters applied to a calculation, which can be modified by DAX functions like CALCULATE.
  10. Create a parameter using "New Parameter" in Power Query, then use it in your queries or measures to create dynamic reports.
