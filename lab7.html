Lab 7: Implementing Row-Level Security

Objective

In this lab, you will learn how to implement Row-Level Security (RLS) in Power BI to restrict data access based on user roles.

Steps

  1. Open your Power BI Desktop file from previous labs.
  2. Go to the Modeling tab and click on "Manage Roles".
  3. Create a new role called "SalesManager".
  4. In the Table filter DAX expression, enter: [Region] = "North"
  5. Create another role called "HR" with the filter: [Department] = "Human Resources"
  6. Click "Save" to create the roles.
  7. Test the roles in Power BI Desktop using "View As Roles".
  8. Publish the report to Power BI Service.
  9. In Power BI Service, go to Security for the dataset.
  10. Assign users or groups to the roles you created.

Exercise

  1. Implement the "SalesManager" and "HR" roles as described in the steps.
  2. Create a new role called "Executive" that has access to all data.
  3. Test all roles in Power BI Desktop to ensure they're working correctly.
  4. Publish your report and set up the security roles in Power BI Service.
  5. Create a matrix visualization that demonstrates how data changes for different roles.
